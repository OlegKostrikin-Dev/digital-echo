# VoltDB: справочник налогоплательщиков

Отдельное хранилище со **справочником налогоплательщиков**: БИН/ИИН, наименование,
**признак резидентности** и служебные статусы. По нему мы определяем, кто из
участников графа — нерезидент (точка входа импорта в модели), с симметрией
всех остальных полей — см. [ADR-002](../decisions/002-voltdb-residency.md).

Поле нерезидентности в самих таблицах ЭСФ использовать не надёжно (много
пропусков и `NULL`), поэтому для расчёта КС опираемся на этот справочник.

## Охват справочника на графе (оценки)

Статистика ниже — для графа порядка **100 тыс. узлов** (типичный масштаб узлов
после построения рёбер из MySQL на одном срезе по датам; доли ориентировочные):

| Категория | Доля |
|---|---|
| Узел найден в справочнике | ~78,5% |
| Из них помечены **нерезидентами** | ~1,3% от всех узлов графа |
| `resident = NULL` (неизвестно) | < 0,001% |
| Узел **не найден** в справочнике | ~21,5% |

Для не найденных в основном: ИП/физлица с отставанием регистрации, снятые БИН,
технические и тестовые идентификаторы. В прототипе такие узлы трактуются
оптимистично как условно резидентные при расчёте — см. [Ограничения](../../docs/limitations.md)
и [ADR-001](../decisions/001-graph-engine.md).

**Семантика `resident`:** `1` — резидент, `0` — нерезидент, `NULL` — статус не
определён; `NULL` **не** считаем нерезидентом.

## Поля справочника, которые используем

Имена колонок в конкретной инсталляции — в коде
`services/backend/src/core/volt_resolver.py`. Логический смысл:

| Роль поля | Тип (типовой) | Смысл |
|---|---|---|
| `tin` | строка фикс. длины | БИН/ИИН **с ведущими нулями** (12 символов) |
| `name_ru` | строка | Наименование на русском |
| `resident` | целое / NULL | Резидентность |
| `state` | целое | Статус налогоплательщика |
| `type` | целое | Тип (юрлицо, ИП и т.д.) |

Остальные десятки колонок справочника в расчёт КС не входят.

## Формат БИН относительно ЭСФ

В MySQL в шапках ЭСФ идентификаторы часто хранятся как **числа**, без ведущих
нулей. В справочнике — **строка с нулями слева**. Перед запросом к справочнику
выполняется нормализация (дополнение до 12 символов). Подробнее —
[Нормализация BIN](bin-normalization.md).
